04 Architect

A web app over the same model.

Architect lets non-coders review the model without editing raw files. It is a view onto the code-first source of truth, not a separate copy: diagrams, traceability, DSM, scenarios, ontology, impact, and DHF all read from the same model.

github.com/memoarchitect/memo-architect
Six modes

Review from many angles

Catalog, Diagram, Action flow, DSM, Scenario, and Ontology modes expose different views without forking the underlying model.

Traceability

Follow the safety thread

DHF workbench

Surface completeness

05 Workbench responsibilities

Same source of truth, lower review friction.

Diagrams

Review architecture, behavior, risk, and interface diagrams built from the same source model.

Trace matrix

Inspect links across intended use, hazards, controls, requirements, behavior, verification, and release evidence.

DSM and impact

Use dependency structure and behavior links to understand change scope before reviews and re-verification planning.

DHF dashboards

Prepare model-backed summaries for design reviews, safety reviews, DHF updates, release baselines, and targeted re-review discussions.
