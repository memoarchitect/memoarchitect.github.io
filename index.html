The evidence decay problem

Trace links are not enough when evidence drifts under change.

Records boundary

Controlled artifacts show relationships.

Requirements, risk files, verification results, reviews, and approvals remain authoritative work products. Impact analysis still needs the architecture semantics behind those links: where a control is implemented, where it acts, and what it covers.

Modeling boundary

SysML is a language, not a medical-device playbook.

Teams still need layers, viewpoints, artifact conventions, rule strengths, workflow, and reusable domain vocabulary before the model can carry reviewer-checkable assurance meaning.

Change boundary

Code feedback stops at repository context.

Git and CI make local change visible. System-level safety impact requires architecture, behavior, risk, and evidence semantics from the model, tied to the exact released baseline being defended.

Repository pillars

Three pillars, one dependency direction.

The organization separates the evidence specification, computable tool runtime, and workbench experience so each pillar can evolve at the right pace.

Evidence mechanisms

MEMO turns drift triggers into computable obligations.

Typed evidence model Ontology + scope

Intended use, hazards, harms, controls, requirements, architecture, behavior, scenarios, and verification evidence have explicit kinds.

>
Closure + consistency Checkable threads

Hazard to control to allocation to behavior step to verification becomes a reviewable, deterministic thread, not a paper-complete link chain.

>
Baseline + freshness Delta evidence

When assumptions, controls, allocations, flows, scenarios, or tests change, MEMO flags evidence that may be stale for the released baseline.

No paper controls.

Risk controls must name where they are implemented and where they act: component, interface contract, behavioral flow step, or scenario.

Re-verification scope becomes defensible.

Structural and behavioral blast radius can drive targeted re-review and re-verification instead of retest-everything or test-too-little.

Documents become compiled views.

Safety and regulatory artifacts can be generated from baselined sources with provenance, stable identifiers, and version stamping.
